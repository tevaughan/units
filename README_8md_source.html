1 # units: Header-only C++ Library for Physical Quantities
2 
3 The vnix::units library defines types and constants that can be used to model
4 any quantity with physical dimension.
5 
6 - no external dependencies needed to build against vnix::units
7  - ruby required to build some header files before installation from
8  git-clone
9 - compiler for C++-14 or later required
10 - interoperability with [Eigen](http://eigen.tuxfamily.org/) supported

41 ## Overview of Library
42 
43 In the present implementation, there are five fundamental dimensions:
44 
45 - length,
46 - mass,
47 - time,
48 - charge, and
49 - temperature.
50 
51 Internally,
52 
53 - vnix::units::dim stores a six-bit rational exponent for each fundamental
54  dimension, and
55 - vnix::units::dimval associates an instance of vnix::units::dim with numeric
56  value.
57 
58 However, the user need not even know about dim and dimval.
59 One may write a simple program, for example, as follows:
60 ```cpp
61 #include <vnix/units.hpp>
62 using namespace std;
63 using namespace vnix::units::flt; // or vnix::units::dbl or vnix::units::ldbl
64 int main() {
65  // Explicit namespace needed for 'time' in order to avoid collision with
66  // function in C standard library.
67  units::flt::time t = 4 * ms; // Multiplication of number by unit.
68  length d = 3.0_km; // Literal for unit.
69  speed v1 = d / t; // Dimension stored in type of v1.
70  dyndim v2 = d / t; // Dimension stored in instance of v2.
71  cout << v1 << endl;
72  return 0;
73 }
74 ```
75 - The output looks like this: `750000 m s^-1`.
76 - See test/dimval-test.cpp for this and other examples.

79 ## Some Details
80 
81 - The library is extensible and reconfigurable via `units.yml`, which briefly
82  defines the system of units.
83  - A ruby script reads `units.yml` and generates
84  - vnix/units.hpp and
85  - vnix/units/dim-base-off.hpp.
86 
87 - Because vnix::units::dimval is a literal type, an instance can be a [constant
88  expression](https://en.cppreference.com/w/cpp/language/constant_expression).
89 
90 - In the vnix::units library, many a standard unit, such as
91 
92  - vnix::units::flt::m for single-precision meter and
93  - vnix::units::dbl::s for double-precision second,
94 
95  is defined as a constant expression.
96 
97 - Many a dimension, such as
98 
99  - vnix::units::flt::length (single precision),
100  - vnix::units::dbl::mass (double precision),
101  - vnix::units::ldbl::force (long-double precision),
102  - vnix::units::flt::current,
103  - etc.,
104 
105  is defined as a *type* that you can use. (This type is a descendant of
106  vnix::units::dimval).
107  ```cpp
108  using namespace vnix::units::flt; // For single-precision types.
109  length l1 = 2.4 * m; // Construct by multiplying unit.
110  length l2 = 7.7_cm; // Construct via user-defined literal.
111  area a = l1 * l2;
112  ```
113 
114 - A variable of such a type knows its dimension without requiring any storage
115  to keep track of the dimension. The compiler keeps track and will find any
116  bug related to a failure of dimensional analysis.
117  ```cpp
118  using namespace vnix::units::dbl; // For double-precision types.
119  area a = 2.4 * m * 3.6; // OOPS! Compiler error: RHS not area!
120  ```
121 
122 - The `auto` keyword in modern C++ allows one not to have to specify the type
123  when declaring a variable, and so a result of multiplication, division, and
124  exponentiation can be stored even if there be no named type for the
125  corresponding dimension. (The type will then be a generic kind of
126  vnix::units::statdim, itself a descendant of vnix::units::dimval.)
127  ```cpp
128  using namespace vnix::units::ldbl; // For long-double types.
129  length foo = 3 * m;
130  auto bar = sqrt(foo); // No simple name for this type.
131  ```
132 
133 - vnix::units::flt::dyndim (or vnix::units::dbl::dyndim or
134  vnix::units::ldbl::dyndim) can be used when the dimension cannot be
135  determined at compile-time, such as when the dimension of a variable is to be
136  read from a file at run-time.
137 
138 - Using memory in the instance to store the dimension, vnix::units::flt::dyndim
139  is unlike, say, vnix::units::flt::length, which does not use memory in the
140  instance to store the dimension; yet vnix::units::flt::dyndim, too, is a
141  descendant of vnix::units::dimval.
142 
143 - The dimension stored in an instance of `dyndim` requires an extra four bytes
144  of storage beyond the number bytes required to store the numeric value of the
145  physical quantity.
146 
147 - vnix::units::sqrt and vnix::units::pow are provided.

150 ## Fetching, Building, and Installing
151 
152 - `clang++-7` and `g++-8.3` are known to work.
153  - Earlier versions might also work, but I have not tested them.
154  - The default is clang++ because the Travis CI build-machine doesn't have a
155  recent enough g++.
156 
157 - To build the library from a clone of the source, be sure that ruby is
158  installed. A couple of the header files are generated from `units.yml`.
159 
160 - See https://github.com/tevaughan/units
161 
162 - To build the test and/or install:
163 
164  ```
165  git clone https://github.com/tevaughan/units.git
166  cd units # Change the working directory into the new clone.
167  vim Makefile # Change PREFIX to specify installation-directory.
168  vim test/Makefile # Change CXX to set compiler.
169  # If Eigen be installed, change EIGEN_DIR to make sure
170  # that the Eigen-compatibility test is compiled and run.
171  make test # Build and run the tests.
172  make doc # Build the documentation via Doxygen.
173  make install # Install the headers to $(PREFIX)/include/vnix.
174  ```
